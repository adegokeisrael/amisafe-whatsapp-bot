# Code of Conduct

## Our Commitment

AmiSafe exists to document and reduce AI-related harm in African communities.
Contributors are expected to uphold that purpose in how they engage with this
project and with each other.

## Expected Behaviour

- Be respectful of contributors and reporters, especially given the sensitive
  nature of the data this project handles.
- Prioritise the privacy and safety of reporters in every code change. If a
  contribution could weaken anonymisation, encryption, or consent guarantees,
  it requires explicit discussion in the pull request before merge.
- Disclose any conflicts of interest relevant to AI harm reporting (e.g.
  employment by a company whose products are commonly reported).

## Unacceptable Behaviour

- Attempting to de-anonymise or re-identify any reporter or report.
- Submitting code intended to enable mass false reporting, denial-of-service,
  or coordinated manipulation of the report database.
- Harassment, discrimination, or exclusionary behaviour of any kind.

## Reporting

If you observe a violation of this Code of Conduct, contact the maintainers
at amisafe@example.org. Reports will be handled confidentially.

## Enforcement

Maintainers reserve the right to remove contributions, reject pull requests,
or ban contributors who violate this Code of Conduct.
